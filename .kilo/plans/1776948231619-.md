# Enemy System Implementation Plan

## Summary
Implement a modular enemy system for the pygame top-down shooter with the following features:
- Spawn function for enemies
- Pathfinding toward the player
- Multiple enemy types (extensible framework)
- Bullet-based damage/kill system
- Clean code organization and comments

## Implementation Details

### 1. Enemy Base Class (`Enemy`)
Generic enemy with health, speed, damage, collision, and visualization.

**Attributes:**
- `pos` (Vector2): World position
- `radius` (int): Collision radius
- `speed` (float): Movement speed
- `health` (int): Current health
- `max_health` (int): Maximum health
- `damage` (int): Damage on player collision
- `color` (tuple): Visual color (RGB)
- `points` (int): Score value

**Methods:**
- `update(dt, player_pos)`: Move toward player each frame
- `take_damage(amount)`: Reduce health, return True if died
- `is_alive()`: Check if health > 0
- `draw(screen, camera_offset)`: Render the enemy

### 2. Enemy Type Registry
Factory pattern using a dictionary to create enemies by type name.

**Types (initial):**
- `basic` - Standard enemy: medium speed, low health, direct charge
- `fast` - High speed, very low health, quick attacks
- `tank` - Low speed, high health, high damage

Each type is a function returning an Enemy instance with specific stats.

### 3. Spawn System
Spawns enemies at regular intervals at random map boundary positions.

**Configuration:**
- `SPAWN_INTERVAL`: Time between spawns (decreases with score)
- `MAX_ENEMIES`: Cap on concurrent enemies
- `SPAWN_POINTS`: Predefined spawn positions along map edges

**Spawn Logic:**
- Pick random edge point from map vertices
- Place enemy just outside the map
- Set target to player position

### 4. Pathfinding
Simple direct pursuit (seek behavior) toward player.

**Behavior:**
- Calculate direction vector: `player_pos - enemy_pos`
- Normalize and multiply by speed * dt
- Move toward player
- For future: add obstacles/waypoints or A*

### 5. Collision & Damage System
**Bullet-Enemy Collision:**
- Check if projectile position is within enemy radius
- Remove projectile on hit
- Apply damage, spawn death effects (score)
- Remove dead enemies

**Player-Enemy Collision:**
- Check if player is within enemy radius + player_radius
- Deal damage to player (future feature)
- Knockback (future)

### 6. Modular File Structure
Split into separate modules for maintainability:

```
game.py                  - Main game loop, setup, rendering
enemy.py                 - Enemy base class and type definitions
spawn_manager.py         - Enemy spawning logic
combat_system.py         - Collision detection, damage, scoring
```

### 7. Code Quality
- Full docstrings for all classes and functions
- Type hints where appropriate
- Consistent naming (snake_case)
- Section headers and comments
- Constants for all tunable values

### 8. Rendering
- Color-coded enemies by type
- Health bar for tanks (future)
- Death particle effects (simple)
- Smooth animation (interpolation)

## Step-by-Step Tasks

1. Create `enemy.py` - Enemy base class and type registry
2. Create `spawn_manager.py` - Spawn logic and interval management
3. Create `combat_system.py` - Collision, damage, projectile interaction
4. Modify `game.py` - Integrate enemy systems, update main loop
5. Add enemy rendering to draw loop
6. Implement bullet-enemy collision detection
7. Test spawn rates, speeds, balance values
8. Verify clean death and removal of enemies

## Future Extensibility

- Subclass Enemy for custom AI behaviors
- Add enemy projectiles
- Add wave-based spawns
- Add boss enemies
- Add pathfinding for obstacles
