## Changes Requested

### 1. Enemies can't go inside the player - Mängija kokkupõrge
**File:** `game.py`
**Location:** After enemy update loop, before collision detection (around line 267)

**Approach:** Add enemy-player collision resolution that pushes enemies away from the player.
After all enemies are updated (moved toward player), check each enemy for overlap with the player. If an enemy is overlapping, push it back so it sits exactly at the edge of the player.

```python
# Enemy-player collision push - Vaenlase tõukumine
for enemy_unit in enemies:
    dist = enemy_unit.pos.distance_to(player_pos)
    min_dist = enemy_unit.radius + player_radius
    if dist < min_dist and dist > 0:
        # Push enemy away from player
        push_dir = (enemy_unit.pos - player_pos).normalize()
        enemy_unit.pos = player_pos + push_dir * min_dist
```

This ensures enemies physically cannot overlap the player - they'll slide around the player instead of going through them.

### 2. Enemies spawn inside the map, not outside - Kaardi sees
**File:** `spawn_manager.py`
**Location:** `_get_random_spawn_point()` method, around line 134-136

**Current behavior:** Line 135-136 offsets the spawn point OUTWARD from the map center by 40 pixels, placing enemies outside the playable area:
```python
outward_dir = (jittered_point - self.world_center).normalize()
spawn_point = jittered_point + outward_dir * 40
```

**Fix:** Change the offset direction to go INWARD instead, placing enemies just inside the map border:
```python
inward_dir = (self.world_center - jittered_point).normalize()
spawn_point = jittered_point + inward_dir * 40
```

This ensures enemies spawn inside the playable area, near the border, and walk toward the player from within the map.
