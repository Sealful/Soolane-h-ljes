## Step-by-Step Changes

### 1. Create `difficulty_manager.py` (new file)
**Purpose:** Track elapsed time and provide difficulty multipliers
- Track `elapsed_time` that increments each frame
- Provide `get_spawn_interval()` - decreases over time (faster spawns)
- Provide `get_enemy_speed_multiplier()` - slight increase over time
- Provide `get_enemy_health_bonus()` - small HP increase over time
- Provide `get_type_weights()` - shifts toward harder types as time increases
- Expose `get_elapsed_time()` for HUD display
- Use config constants at top for easy tuning (e.g., `DIFFICULTY_RAMP_RATE`)
- Extensible: new difficulty hooks can be added as methods

**Key values:**
- `BASE_SPAWN_INTERVAL = 2.0` → scales down to min `0.5s`
- `SPEED_MULTIPLIER` starts at `1.0`, ramps to ~`1.5x` over 5 minutes
- `HEALTH_BONUS` starts at `0`, adds ~`+2 HP` over 5 minutes

### 2. Create `vfx_manager.py` (new file)
**Purpose:** Particle system for surface hit visual effects
- `Particle` class: position, velocity, lifetime, max_lifetime, color, size
- `VFXManager` class: manages active particle effects
- `spawn_hit_effect(position, effect_type)` - creates particles at hit location
- Effect types:
  - `"enemy"` - red/orange burst particles (when enemy hit)
  - `"wall"` - white spark particles (when bullet hits map border)
  - `"wall"` - blue spark particles for map border collisions
- Each effect type defines particle count, color palette, spread angle, speed range
- `update(dt)` - ages particles, removes expired ones
- `draw(screen, camera_offset)` - renders all active particles
- **Registry pattern** for effect types: `_effect_types = {}` with `register_effect_type(name, config)` so new effects can be added without modifying core logic

### 3. Modify `enemy.py` - Increase sizes, remove score references
- **Size changes:**
  - `basic`: radius `14` → `18`
  - `fast`: radius `10` → `14`
  - `tank`: radius `22` → `28`
- **Remove `points` field** from base Enemy class (no longer needed)
- Remove `points` from all factory functions (`_make_basic`, `_make_fast`, `_make_tank`)
- Keep everything else intact (extensibility preserved)

### 4. Modify `spawn_manager.py` - Integrate difficulty
- Accept optional `difficulty_manager` reference in constructor
- If present, use `difficulty_manager.get_spawn_interval()` instead of hardcoded `SPAWN_INTERVAL`
- Use `difficulty_manager.get_type_weights()` to select harder enemy types over time
- Keep fallback to static config if no difficulty manager provided

### 5. Modify `game.py` - Major refactor
#### 5a. Remove scoring system
- Remove `score = 0` variable
- Remove `score += enemy_unit.points` in collision code
- Remove score rendering (`font.render("Score: ...")`)
- Remove `points` references from enemy collision logic

#### 5b. Add time system + difficulty
- Add `game_time = 0` tracker, increment each frame with `dt`
- Instantiate `DifficultyManager`
- Instantiate `VFXManager`
- Pass difficulty manager to `SpawnManager`
- Use difficulty scaling in enemy spawning

#### 5c. Add drift to player movement
- Replace direct position changes with velocity-based system:
  ```python
  player_velocity = pygame.Vector2(0, 0)
  player_acceleration = 800     # acceleration rate
  player_drag = 5.0             # friction coefficient (higher = less drift)
  ```
- Instead of `player_pos.x += player_speed * dt`, use:
  ```python
  input_dir = pygame.Vector2(0, 0)
  if keys[K_w]: input_dir.y -= 1
  if keys[K_s]: input_dir.y += 1
  if keys[K_a]: input_dir.x -= 1
  if keys[K_d]: input_dir.x += 1
  
  if input_dir.length() > 0:
      input_dir = input_dir.normalize()
      player_velocity += input_dir * player_acceleration * dt
  
  # Apply drag (friction)
  player_velocity *= (1 - player_drag * dt)
  
  # Apply velocity to position
  player_pos += player_velocity * dt
  ```
- This creates smooth acceleration/deceleration (drift feel)

#### 5d. Slower shooting
- `shoot_cooldown: 0.12` → `0.25`

#### 5e. VFX integration
- On bullet-enemy hit: `vfx_manager.spawn_hit_effect(enemy.pos, "enemy")`
- On bullet-wall hit (projectile removed): detect which boundary was hit, spawn `"wall"` effect at exit point
- Call `vfx_manager.update(dt)` each frame
- Call `vfx_manager.draw(screen, camera_offset)` in render section

#### 5f. HUD update
- Replace score display with elapsed time display
- Format: `Time: MM:SS` or `Time: M:SS`
